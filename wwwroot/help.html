üîê Understanding Authentication: 2-Legged vs 3-Legged Tokens

Before you start building, you need to understand authentication ‚Äî the key that unlocks APS APIs. Think of authentication as getting the right badge to enter different areas of a building.

2-Legged Token (Server-to-Server)

GET /auth/token
What it is:

A 2-legged token authenticates your application directly with Autodesk. It's like a master key that your app uses to access data.

When to use it:
  • ‚úÖ Automated processes (no human involved)
  • ‚úÖ Backend services and integrations
  • ‚úÖ Reading public or shared data
  • ‚úÖ System-to-system data synchronization
  • ‚úÖ Scheduled tasks and batch jobs
Example use cases:
  • üìä A nightly job that generates reports from ACC data
  • üîÑ Syncing project data to your company's data warehouse
  • üìà A dashboard that displays project statistics
  • ü§ñ Automated file processing and conversions
üí° Key Point: 2-legged tokens represent your application, not a specific user. Use this when your app needs to work independently.

3-Legged Token (User Authorization)

GET /auth/login
What it is:

A 3-legged token authenticates a specific user who logs in with their Autodesk credentials. It's like a personalized badge that shows who is accessing the data.

When to use it:
  • ‚úÖ Actions performed by a specific user
  • ‚úÖ User-specific data access (their projects, their files)
  • ‚úÖ Web applications where users log in
  • ‚úÖ Operations that require user consent
  • ‚úÖ Audit trails showing who did what
Example use cases:
  • üë§ A web portal where users manage their own projects
  • üìù Applications that create files on behalf of a user
  • ‚úèÔ∏è Tools that edit documents (needs user permission)
  • üîí Access to private or sensitive project data
‚ö†Ô∏è Important: 3-legged tokens represent a specific user. Use this when actions need to be attributed to a person, or when accessing user-specific data.

üìã Step-by-Step Workflow

1 Get an Access Token

First, you need a "token" - think of it as a temporary password that proves you're allowed to access the data.

  • Go to section 1. Authentication üîê
  • Click on GET /auth/token (it says "2-Legged Token")
  • Click the blue "Execute" button
  • Copy the access_token from the response (it's a long string of letters and numbers)
üí° Tip: The token expires after 1 hour. If you get "Unauthorized" errors later, come back and get a fresh token!
2 Get Your Hubs

A "Hub" is your BIM 360 or ACC account.

  • Go to section 2. Hubs üèóÔ∏è Data Management API
  • Click on GET /hubs
  • Click "Try it out"
  • Paste your token in the token field
  • Click "Execute"
  • Copy a hubId from the response (looks like "b.3def0ac0-6276...")
‚ö†Ô∏è Important: The hubId MUST start with "b." - don't remove this prefix!
3 Get Projects in Your Hub

Projects are the individual construction projects within your account.

  • Go to section 3. Projects üìÅ Data Management API
  • Click on GET /projects
  • Click "Try it out"
  • Paste your token
  • Paste your hubId (with the "b." prefix!)
  • Click "Execute"
  • Copy a projectId from the response
4 Explore Further!

Now you can explore folders, users, companies, and more using your token, hubId, and projectId!

‚ùå Common Errors & Solutions

Error: "Unauthorized" (401)

Problem: Your token expired or is invalid
Solution: Go back to GET /auth/token and get a fresh token!

Error: "Forbidden" (403)

Problem: Your token doesn't have permission
Solution: Make sure your APS app has all APIs enabled in the APS Portal

Error: "Bad Request" (400)

Problem: Missing or wrong "b." prefix
Solution:
  • For Hubs/Projects/Folders: ADD "b." prefix
  • For Companies/Users/Roles: We auto-remove it, so either way works!

Error: "Not Found" (404)

Problem: The resource doesn't exist or wrong ID
Solution: Double-check your IDs - copy them exactly from previous API responses

Example Code: 
var client = new HttpClient();
var request = new HttpRequestMessage(
    HttpMethod.Post, 
    "https://developer.api.autodesk.com/
    authentication/v2/token"
);

    new {
        client_id = "your_id",
        client_secret = "your_secret",
        grant_type = "client_credentials"
    });
var response = await client.SendAsync(request);
var token = await response.Content
    .ReadAsStringAsync();
